TailwindCss

Qu'est ce que TailwindCss

TailwindCss est un framework CSS qui permet de designer facilement et simplement les pages d'un site web. Il s'agit d'un framework utility first (il s'git d'une méthode de travail en CSS pour organiser les éléments, faciliter la la maintenance et limiter les effets de bords en donnant des noms aux classes les noms qui correspondent aux propriétés CSS ) créer en Août 2017 par Adam Wathan. Ce framework ne contient que les bases brutes comme les marges, les tailles, les styles, les couleurs, le positionnement, etc, ce qui lui permet de donner une certaine liberté au développeur lors de la création de la création de ses éléments.

Avantages

  • Un outil assez personnalisable

    En effet cette librairie a été conçue afin de donner une certaine liberté aux développeurs. Le fichier de configuration (tailwind.config.js) est personnalisable et il est possible d'y ajouter des classes personnelles.
    Par exemple avec les éléments gold qui ont été ajouté, il est possible de les apppeler pour une couleur de police par exemple en faisant: text-gold-light. Il est également possible de personnaliser un élément son nom de classe-[propriété] comme text-[50px].

  • Une prise en main rapide

    La librairie tailwind contient des classes intuitives et la documentation fournit par le site officiel. Il y a aussi une palette de couleurs conséquente avec 8 couleurs et 10 déclinaisons pour chaque couleur. Par exemple comme vous pouvez le voir dans le code suivant, bg-cyan-400 (bg pour background) permet de donner la couleur cyan qu'on a pour notre barre en haut de page. Le code text-3xl équivaut à font-size: 1.875rem; line-height: 2.25rem;. Il s'agit d'une valeur de base et il est possible de la trouver dans la documentation de tailwind.

  • Ne contient pas de composant standards

    Contrairement à ses concurents directs Boostrap et de Foundation , il ne permet pas de creer des éléments tels que des bouttons ou les barres de navigations. Cependant cette contrainte donne une extrême liberter de personnalisation aux développeurs lors de a création de ses éléments et évite de combattre le framework pour imposer son style.

Inconvénients

  • Une conception plus importante

    Sur des projets très importants avec un grand nombre de composant, Boostrap reste beaucoup plus adapté. Tailwind ne permet que de styliser que les éléments mais pas d'en créer, de ce fait il est necessaire que le développeur créé tous les éléments lui même.

  • Un code assez vite illisible

    Ceci est un point commun avec tous les framework css. Dès qu'il y a un très grand nombre de classe pour un élément, il devient difficile de lire et de maintenir le code.
